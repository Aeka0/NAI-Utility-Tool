# Security Policy

## Supported Versions

当前仓库仍在快速迭代阶段，默认只对最新主分支上的代码进行安全修复与响应。

## Reporting a Vulnerability

如果你发现了以下问题，请不要直接公开提交 issue：

- API Token、账户信息或凭证泄露
- 可导致本地敏感文件泄露的问题
- 可被利用执行任意代码、任意文件覆盖或路径穿越的问题
- 其他你认为不适合公开披露的安全缺陷

建议通过 GitHub 的 [Private Security Reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability) 功能提交（仓库页 → Security 标签页 → "Report a vulnerability"），或通过其他私下渠道联系维护者。上报时请尽量附上：

- 问题描述
- 影响范围
- 复现步骤
- 可能的修复方向

在修复发布前，请避免公开披露细节、PoC、敏感日志或截图。
